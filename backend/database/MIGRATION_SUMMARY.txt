## 🔄 Migration Process

1. **Drop Foreign Keys** - Temporarily removes all FK constraints
2. **Create Mappings** - Generates UUID for each existing integer ID
3. **Add Temp Columns** - Creates *_uuid columns
4. **Populate UUIDs** - Maps old IDs to new UUIDs
5. **Replace Columns** - Drops integer columns, renames UUID columns
6. **Recreate FKs** - Restores all foreign key relationships

## 📈 Benefits of UUIDs

✅ **Globally unique** - No collisions across systems
✅ **Distributed-friendly** - Generate IDs anywhere
✅ **Security** - Non-sequential, harder to guess
✅ **Merge-friendly** - Easy to combine databases
✅ **Offline support** - Generate IDs without DB connection
