=================================================================
  UUID MIGRATION - QUICK START GUIDE
=================================================================

⚡ FAST TRACK (5 Minutes)
=================================================================

STEP 1: BACKUP (1 minute)
--------------------------
Open phpMyAdmin → Select "global_pos" → Click "Export" → Click "Go"
Save the file as: backup_before_uuid.sql


STEP 2: CHECK STATUS (30 seconds)
----------------------------------
Open browser:
http://localhost/backend/database/check_uuid_status.php

Look at the summary:
- If "Integer Columns" > 0 → Continue to Step 3
- If "Integer Columns" = 0 → Already done! ✅


STEP 3: RUN MIGRATION (2-3 minutes)
------------------------------------
Click the green button "🚀 Start UUID Migration"
OR visit:
http://localhost/backend/database/migrate_to_uuid.php

Wait for: "=== MIGRATION COMPLETED SUCCESSFULLY ==="
DO NOT close browser during migration!


STEP 4: VERIFY (30 seconds)
----------------------------
Go back to:
http://localhost/backend/database/check_uuid_status.php

Confirm:
✅ "Integer Columns" = 0
✅ "UUID Columns" = (some number > 0)
✅ Foreign Keys section shows constraints


STEP 5: TEST APP (1 minute)
----------------------------
1. Login to your POS application
2. Create a test order
3. View products list
4. If everything works → SUCCESS! 🎉

=================================================================

⚠️ IF SOMETHING GOES WRONG
=================================================================

OPTION 1: Automatic Rollback
-----------------------------
The migration uses transactions. If it fails, changes are
automatically rolled back. Just try again.


OPTION 2: Manual Restore
-------------------------
In phpMyAdmin:
1. Select "global_pos" database
2. Click "Import"
3. Choose your backup file
4. Click "Go"
